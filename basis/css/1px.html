移动端1px解决方案 ​

1. 伪元素 + transform缩放 ​

基础实现 ​

css
.border-1px {
  position: relative;
}
.border-1px::after {
  content: "";
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 1px;
  background: #000;
  transform: scaleY(0.5);
  transform-origin: 0 0;
}

媒体查询适配方案（推荐） ​

css
/* 默认情况（设备像素比1-1.9） */
.border-1px::after {
  transform: scaleY(0.5);
}

/* 设备像素比2-2.9 */
@media (-webkit-min-device-pixel-ratio: 2) {
  .border-1px::after {
    transform: scaleY(0.5);
  }
}

/* 设备像素比3+ */
@media (-webkit-min-device-pixel-ratio: 3) {
  .border-1px::after {
    transform: scaleY(0.333);
  }
}

媒体查询必要性说明 ​

  1. 非必需但推荐：基础方案在大多数设备上表现良好
  2. 精确适配场景：
    • 超高分辨率设备（如4K屏手机）
    • 需要严格保证1px视觉效果
    • 不同设备需要不同缩放比例
  3. 兼容性写法：
css
/* 兼容性写法 */
@mixin scale-border($direction: bottom) {
  @media (-webkit-min-device-pixel-ratio: 1.5) {
    transform: scaleY(0.7);
  }
  @media (-webkit-min-device-pixel-ratio: 2) {
    transform: scaleY(0.5);
  }
  @media (-webkit-min-device-pixel-ratio: 3) {
    transform: scaleY(0.33);
  }
}

四边框实现 ​

css
.border-all-1px {
  position: relative;
}
.border-all-1px::after {
  content: "";
  position: absolute;
  top: 0;
  left: 0;
  width: 200%;
  height: 200%;
  border: 1px solid #000;
  transform: scale(0.5);
  transform-origin: 0 0;
  box-sizing: border-box;
  pointer-events: none;
}

优点：实现简单，兼容性好
缺点：需要额外元素或伪元素

2. viewport + rem方案 ​

核心原理 ​

通过设置viewport的scale=0.5，使CSS中的1px等于物理像素的1px，同时配合rem适配不同屏幕。

完整实现步骤 ​

  1. 设置viewport meta标签
html
<meta name="viewport" content="width=device-width,initial-scale=0.5,maximum-scale=0.5,minimum-scale=0.5,user-scalable=no">
  1. 动态计算rem基准值
javascript
// 在HTML头部添加
document.documentElement.style.fontSize = 
  document.documentElement.clientWidth / 10 + 'px';
  1. CSS中使用rem单位
css
/* 假设设计稿宽度为750px */
.element {
  width: 1rem;  /* 实际显示75px */
  height: 0.5rem;
  border: 1px solid #000; /* 真实的1物理像素 */
}

适配方案对比表 ​

方案优点缺点适用场景
纯viewport缩放实现简单文字会变小简单页面
viewport+rem精确控制需要JS计算复杂项目
viewport+vw无需JS兼容性要求高现代浏览器项目

注意事项 ​

  1. 文字大小问题：需要单独设置body字体大小
css
body {
  font-size: 24px; /* 实际显示12px */
}

  1. 第三方组件适配：可能需要重置组件的样式基准

  2. 图片模糊问题：需要准备@2x和@3x图片资源

  3. 调试技巧：Chrome设备模式中勾选"Show media queries"

优点：直接解决物理像素问题，一劳永逸
缺点：全局影响较大，需要全面适配rem单位

3. border-image方案 ​

css
.border-image-1px {
  border-width: 1px 0 0;
  border-image: url('data:image/png;base64,...') 2 0 stretch;
}

优点：可以实现复杂边框
缺点：修改颜色麻烦，需要生成图片

4. box-shadow方案 ​

css
.box-shadow-1px {
  box-shadow: 0 1px 1px -1px rgba(0, 0, 0, 0.5);
}

优点：代码简单
缺点：颜色控制不精确，有模糊效果

5. SVG方案 ​

css
.svg-1px {
  background: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' height='1' width='100%'><line x1='0' y1='0' x2='100%' y2='0' stroke='#000' stroke-width='1'/></svg>");
}

优点：矢量图形清晰
缺点：代码较长，修改不便

综合对比 ​

方案兼容性灵活性实现难度推荐度

viewport缩放中差难★★☆☆☆

最佳实践建议：优先使用伪元素+transform方案
